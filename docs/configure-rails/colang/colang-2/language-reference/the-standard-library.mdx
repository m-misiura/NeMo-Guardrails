<a id="the-standard-library"></a>


The Colang Standard Library (CSL) provides an abstraction from the underlying event and action layer and offers a semantic interface to design interaction patterns between the bot and the user. Currently, there are the following library files available under `nemoguardrails/colang/v2_x/library/` ([GitHub link](https://github.com/NVIDIA-NeMo/Guardrails/tree/develop/nemoguardrails/colang/v2_x/library)):

<Cards>

<Card title="Core Flows" href="/configure-guardrails/colang/colang-2/language-reference/the-standard-library/core">

Core user, bot, wait, and notification flows.

</Card>

<Card title="Timing Flows" href="/configure-guardrails/colang/colang-2/language-reference/the-standard-library/timing">

Wait, timer, and silence detection flows.

</Card>

<Card title="LLM Flows" href="/configure-guardrails/colang/colang-2/language-reference/the-standard-library/lmm">

LLM continuation and utility flows.

</Card>

<Card title="Avatar Flows" href="/configure-guardrails/colang/colang-2/language-reference/the-standard-library/avatars">

Gesture, posture, and avatar-related flows.

</Card>

<Card title="Guardrail Flows" href="/configure-guardrails/colang/colang-2/language-reference/the-standard-library/guardrails">

Standard guardrail flow definitions.

</Card>

<Card title="Attention Flows" href="/configure-guardrails/colang/colang-2/language-reference/the-standard-library/attention">

User attention tracking and checks.

</Card>

</Cards>
To use the flows defined in these libraries you have two options:

- [Recommended] Import the standard library files using the import statement: e.g. `import llm`
- Copy the corresponding `*.co` file directly inside your Colang script directory.

Note that the `import <library>` statement will import all available flows of the corresponding library.
